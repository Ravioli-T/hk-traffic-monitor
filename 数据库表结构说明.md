# HK Traffic Monitor — 数据库表结构说明

>
>
> 数据库: MySQL 8.0 · 库名: `hk_traffic` · 连接: 见 `.env` 文件

---

## 1. traffic_readings — 交通读数（核心数据表）

每一行 = 某个检测器的某条车道在某个 30 秒时段的一次测量。

| 字段 | 类型 | 说明 | 示例值 |
|------|------|------|--------|
| `id` | BIGINT, 自增主键 | 行 ID | 1, 2, 3... |
| `detector_id` | VARCHAR(20) | 检测器编号 | `AID01101`, `AID20051` |
| `source_type` | ENUM | 数据来源：`strategic`（战略道路）或 `lamppost`（智慧灯柱） | `strategic` |
| `timestamp` | DATETIME | 测量时间（30 秒精度） | `2026-03-14 08:30:00` |
| `lane_id` | VARCHAR(30) | 车道名称 | `Fast Lane`, `Middle Lane`, `Slow Lane`, `Middle Lane 1`, `Middle Lane 2` |
| `speed` | INT | 车速 (km/h) | 67 |
| `volume` | INT | 30 秒内通过的车辆数 | 4 |
| `occupancy` | INT | 道路占用率 (%) | 12 |
| `speed_sd` | DECIMAL(5,2) | 速度标准差 | 14.60 |
| `valid` | CHAR(1) | 数据有效性：`Y`=检测器在线, `N`=离线 | `Y` |
| `created_at` | TIMESTAMP | 入库时间（自动生成） | `2026-03-18 01:00:00` |

**关键索引：**
- `idx_detector_time (detector_id, timestamp)` — 按检测器+时间查询（ML 训练的主要查询模式）
- `idx_timestamp (timestamp)` — 按时间范围查
- `idx_source_type (source_type)` — 按来源筛选

**车道说明：** 不同检测器的车道数不同（1-4 条），同一检测器在同一时刻会有多行记录（每条车道一行）。

| 车道数 | lane_id 取值 |
|--------|-------------|
| 1 车道 | Fast Lane |
| 2 车道 | Fast Lane, Slow Lane |
| 3 车道 | Fast Lane, Middle Lane, Slow Lane |
| 4 车道 | Fast Lane, Middle Lane 2, Middle Lane 1, Slow Lane |

## 2. detector_info — 检测器元数据

每一行 = 一个物理检测器的位置和归属信息。从官方 CSV 一次性导入。

| 字段 | 类型 | 说明 | 示例值 |
|------|------|------|--------|
| `detector_id` | VARCHAR(20), 主键 | 检测器编号（与 traffic_readings 关联） | `AID01101` |
| `district` | VARCHAR(50) | 所属区域（香港 18 区之一） | `Kowloon City` |
| `road_name_en` | VARCHAR(200) | 道路英文名 | `Lion Rock Tunnel Road` |
| `road_name_tc` | VARCHAR(200) | 道路中文名 | `獅子山隧道公路` |
| `latitude` | DECIMAL(9,6) | 纬度 | 22.345600 |
| `longitude` | DECIMAL(9,6) | 经度 | 114.178900 |
| `direction` | VARCHAR(50) | 检测方向 | `South East` |
| `easting` | DECIMAL(12,2) | HK1980 格网 X 坐标 | 841163.00 |
| `northing` | DECIMAL(12,2) | HK1980 格网 Y 坐标 | 819561.00 |
| `source_type` | ENUM | `strategic` 或 `lamppost` | `strategic` |
| `created_at` | TIMESTAMP | 入库时间 | `2026-03-18 00:00:00` |

**数据规模：** strategic 约 400+ 个检测器（覆盖全港 18 区），lamppost 约 17 个（集中在少数试点区域）。

## 3. 两张表的关系

```
detector_info.detector_id  ←——→  traffic_readings.detector_id
      (1)                              (N)
  一个检测器                       多条交通读数
```

关联查询示例：查某个区域所有检测器的最近读数：

```sql
SELECT r.*, d.district, d.road_name_en, d.latitude, d.longitude
FROM traffic_readings r
JOIN detector_info d ON r.detector_id = d.detector_id
WHERE d.district = 'Kowloon City'
  AND r.timestamp >= '2026-03-14 08:00:00'
  AND r.valid = 'Y'
ORDER BY r.timestamp DESC;
```

## 5. 对 ML 训练有用的查询

### Isolation Forest（RQ1 异常检测）

```sql
-- 取某检测器全部 valid 数据，用于训练
SELECT timestamp, lane_id, speed, volume, occupancy, speed_sd
FROM traffic_readings
WHERE detector_id = 'AID01101' AND valid = 'Y'
ORDER BY timestamp;
```

### Prophet（RQ2 流量预测）

```sql
-- Prophet 需要 ds + y 两列，取某检测器某车道的 volume 时间序列
SELECT timestamp AS ds, volume AS y
FROM traffic_readings
WHERE detector_id = 'AID01101'
  AND lane_id = 'Fast Lane'
  AND valid = 'Y'
ORDER BY timestamp;
```

### Strategic vs Lamppost 对比（RQ3）

```sql
-- 按来源统计 valid 率
SELECT source_type,
       COUNT(*) AS total,
       SUM(valid = 'Y') AS valid_count,
       ROUND(SUM(valid = 'Y') / COUNT(*) * 100, 2) AS valid_rate
FROM traffic_readings
GROUP BY source_type;

-- 按来源 + 区域统计检测器数
SELECT d.source_type, d.district, COUNT(*) AS detector_count
FROM detector_info d
GROUP BY d.source_type, d.district
ORDER BY d.source_type, detector_count DESC;
```

## 6. 注意事项

1. **valid='N' 的数据**：表示检测器离线，speed/volume/occupancy 通常为 0，训练模型时建议过滤掉（`WHERE valid = 'Y'`）。
2. **时间粒度**：原始数据为 30 秒一条，如果模型需要更粗的粒度（如 5 分钟、15 分钟），在查询时做聚合即可。
3. **多车道处理**：同一检测器同一时刻有多行（每车道一行）。可以按需选择单车道训练，或先聚合（如取各车道速度的均值）。
4. **历史数据**：目前已导入 2026-03-14 一整天的数据（约 140 万行）。如需更多日期，下载后放到 `data/raw/history/` 运行导入脚本即可。
