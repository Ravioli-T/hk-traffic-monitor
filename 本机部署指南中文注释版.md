# Hong Kong Smart Traffic Monitoring System

告诉你如何在拉取项目后成功跑起来

## Architecture Overview

The system is organized into four main layers:

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  Layer 1: Ingestion                                                          │
│  Fetch XML from HK Gov APIs (Strategic Roads + Smart Lampposts)               │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  Layer 2: MQTT Broker (Eclipse Mosquitto, localhost:1883)                     │
│  Publish parsed data → Subscribe & persist                                   │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  Layer 3: MySQL 8.0                                                          │
│  Store real-time + historical traffic readings, detector metadata            │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  Layer 4: ML API + Frontend                                                  │
│  FastAPI (Isolation Forest, Prophet) → React + Leaflet + ECharts              │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Quick Start

先安装好mysql和eclipse mosquitto，版本如下，mosquitto下最新版即可

### Prerequisites

- Python 3.10+
- MySQL 8.0
- Eclipse Mosquitto 2.x (MQTT broker on `localhost:1883`)

### Installation

python环境配置

```bash
pip install -r requirements.txt
```

### Configuration

env这里用的数据库配置，需要根据scripts文件夹下的create_local_db来建库、建用户和建表  
启动mysql连接到本地数据库，然后运行一下create_local_db.sql，即可直接使用现有的env  

1. Copy `.env.example` to `.env`
2. Set database credentials and other environment variables

安装完mqtt后，用管理员权限修改一下Mosquitto文件夹  
（我直接装在C盘了，路径就是C:\Program Files\Mosquitto）中的mosquitto.conf，  
末尾加上以下两行：  
```bash
listener 1883  
allow_anonymous true  
```
然后在系统搜索“服务”中重启一下Mosquitto Broker服务  
这是允许匿名监听，不然mqtt模块会无法运行，需要自行额外配置权限，更麻烦  

### Database Setup

这里的detector info从gov网站下载，  
道路交通检测器 https://data.gov.hk/en-data/dataset/hk-td-sm_4-traffic-data-strategic-major-roads/resource/4d839504-8d82-4b46-87a2-70966b27c6cf  
智能路灯 https://data.gov.hk/en-data/dataset/hk-td-tis_33-traffic-data-traffic-detectors-installed-at-smart-lampposts/resource/b003463c-6c80-4fd1-9d18-f2d8c8240a3b  
记得放到data/raw里之后改名为strategic_detector_info和lamppost_detector_info
这是道路检测器的位置信息  

```bash
python scripts/init_db.py
python scripts/import_detector_info.py
```

### 端到端测试
做完了上面这些配置之后，可以运行一下scripts里的e2e.py，确认各个环节和连接没有问题

### 现有的导入历史数据功能
https://data.gov.hk/en-data/dataset/hk-td-sm_4-traffic-data-strategic-major-roads/resource/78f89483-8e15-4186-9533-b562d0fca6f2
在这个页面找到 Download Historical Data，即可按照日期下载某一天的历史数据。   
将下载的一天的数据文件夹重命名为形如以下的格式  
```bash
Traffic Speed, Volume and Road Occupancy (Raw Data) - 20260314
```
放到data/raw/history目录下，然后使用scripts中的import_historical_data来导入，导入方法在该脚本中有具体说明  

### Run the Pipeline 这个还没做，不用按照这个步骤来

```bash
# Real-time mode: fetch → publish → subscribe → store
python main.py --mode realtime

# Import historical data (after placing files in data/raw/)
python main.py --mode import
```

## Folder Structure

| Directory | Purpose |
|-----------|---------|
| `config/` | Central configuration (settings, env loading) |
| `data/` | Local data storage (raw downloads, processed files) |
| `scripts/` | One-off scripts (DB init, detector import, historical import) |
| `src/` | Core application code (fetcher, mqtt, database, pipeline, ml) |
| `frontend/` | React app (separate teammate responsibility) |
| `notebooks/` | Jupyter notebooks for EDA |
| `tests/` | Pytest test suite |
| `logs/` | Runtime log files |
